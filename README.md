# Smart Home Case — система управления умным домом (IoT)

Домен: IoT / Smart Home  
Тип проекта: дипломный проект  
Роль: системный аналитик 

## 📌 Контекст и проблема
Рынок решений для умного дома активно развивается, однако многие существующие приложения фокусируются на управлении устройствами и слабо учитывают реальные сценарии использования жилья и вопросы энергосбережения.

В ходе предварительного анализа было выявлено, что пользователи:
- не понимают, какие устройства потребляют больше всего энергии  
- не имеют удобных инструментов для управления энергопотреблением  
- сталкиваются со сложными и перегруженными интерфейсами  

Заказчику было важно получить решение, которое:
- централизует управление устройствами умного дома  
- помогает снижать энергопотребление за счёт сценариев и аналитики энергопотребления
- остаётся понятным для конечных пользователей и прозрачным для бизнеса  

## 🎯 Цель проекта
Спроектировать систему управления умным домом с фокусом на энергосбережение и подготовить полный комплект аналитической документации для разработки MVP мобильного приложения.

## 👨‍💻 Моя роль
В проекте выступал в роли системного аналитика:
- сбор и анализ требований  
- формирование пользовательских сценариев  
- моделирование системы  
- подготовка аналитических артефактов для разработки и согласования со стейкхолдерами  

Проект выполнялся с постепенным увеличением ответственности: от анализа отдельных требований до формирования полного пакета документации.

## 📋 Работа с требованиями
Проведено:
- интервью с заказчиками для выявления бизнес-целей и ограничений  
- интервью с потенциальными пользователями для понимания реальных сценариев использования  

На основе интервью:
- сформированы ключевые пользовательские сценарии  
- определён объём MVP  
- описаны функциональные требования:
  - управление устройствами  
  - активация режима энергосбережения  
  - мониторинг и аналитика энергопотребления  

## 🧩 Моделирование и проектирование системы
Подготовлены:
- DFD-диаграммы — потоки данных между пользователями, мобильным приложением, устройствами и внешними компонентами  
- ER-диаграмма — модель данных системы  
- Словарь данных — описание сущностей, атрибутов и связей  

Моделирование позволило:
- структурировать требования  
- выявить зависимости между компонентами  
- заложить основу для масштабирования решения
